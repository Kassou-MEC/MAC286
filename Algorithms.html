Complexity of Algorithms in Java

Understanding Time and Space Complexity with Java Examples

What are Algorithms?

An algorithm is a step-by-step procedure to solve a problem or accomplish a task. In computer science, algorithms are the foundation of programming - they provide the logic that manipulates data stored in data structures.

Characteristics of Good Algorithms

  • Correctness: It should produce the correct output for all valid inputs
  • Efficiency: It should use minimal computational resources (time and memory)
  • Clarity: It should be easy to understand and implement
  • Robustness: It should handle edge cases and unexpected inputs gracefully

Why Algorithm Analysis Matters

Analyzing algorithms helps us understand how they perform as the input size grows. This is crucial for writing efficient programs, especially when working with large datasets.

Types of Complexity

There are two main types of complexity:

  1. Time Complexity: Measures the amount of time an algorithm takes to complete as a function of the input size.
  2. Space Complexity: Measures the amount of memory an algorithm uses as a function of the input size.

What Does Input Size Represent?

The input size can vary depending on the problem:

  • For a list of numbers, the input size is the number of elements in the list.
  • For a matrix, the input size could be the number of rows and columns.
  • For a string, the input size is the length of the string.
  • For a graph, the input size could be the number of vertices and edges.

Understanding Big O Notation

Big O notation is a mathematical concept used in computer science and mathematics to describe the upper bound of an algorithm's time complexity or space complexity in terms of the input size.

Common Time Complexities

Here are the most common time complexities you'll encounter, from fastest to slowest:

  • O(1) - Constant Time: Runtime doesn't depend on input size
  • O(log n) - Logarithmic Time: Runtime grows logarithmically with input size
  • O(n) - Linear Time: Runtime grows proportionally with input size
  • O(n log n) - Linearithmic Time: Runtime grows slightly faster than linear
  • O(n²) - Quadratic Time: Runtime grows proportionally to the square of input size
  • O(2ⁿ) - Exponential Time: Runtime doubles with each additional input element

Execution Time Formula

The execution time of a program or algorithm can be calculated using the following formula:

\[ \text{Execution Time} = \text{Number of CPU Cycles} \times \text{Cycle Time} \]

Where:

  • Number of CPU Cycles: The total number of clock cycles required to execute the program
  • Cycle Time: The time taken to complete one clock cycle, which is the inverse of the clock speed

\[ \text{Cycle Time} = \frac{1}{\text{Clock Speed}} \]

Example Calculation

Problem: A program requires 10,000 clock cycles to execute. The processor has a clock speed of 2 GHz.

Solution:

\[ \text{Cycle Time} = \frac{1}{2 \times 10^9} = 0.5 \times 10^{-9} \text{ seconds} = 0.5 \text{ nanoseconds} \]

\[ \text{Execution Time} = 10,000 \times 0.5 \times 10^{-9} = 5 \times 10^{-6} \text{ seconds} = 5 \text{ microseconds} \]

O(1) - Constant Time

The runtime is constant and does not change with the size of the input.

Examples of O(1) Operations

  • Accessing an element in an array by index
  • Inserting or deleting an element at the beginning of a linked list (if you have a pointer to the head)
  • Pushing or popping from a stack
  • Accessing a value in a hash table (on average)
Java O(1) Example: Array Access 
public class ConstantTimeExample {
    public static void main(String[] args) {
        // Array with 5 elements
        int[] arr = {10, 20, 30, 40, 50};
        
        // Accessing an element by index - O(1) operation
        int index = 2;
        int element = arr[index];
        
        System.out.println("Element at index " + index + ": " + element);
    }
}

Execution Time Calculation for Array Access

For a MacBook Pro M4 (4.4 GHz CPU), accessing an array element (O(1)) takes approximately:

\[ \text{Cycle Time} = \frac{1}{4.4 \times 10^9} = 2.27 \times 10^{-10} \text{ seconds} \]

\[ \text{Execution Time} = 5 \text{ cycles} \times 2.27 \times 10^{-10} = 1.14 \times 10^{-9} \text{ seconds} = 1.14 \text{ nanoseconds} \]

O(log n) - Logarithmic Time

The runtime grows logarithmically with the size of the input. Each step of the algorithm reduces the problem size by a significant factor.

Examples of O(log n) Operations

  • Binary search in a sorted array
  • Finding an element in a balanced binary search tree
  • Euclidean algorithm for finding GCD
Java O(log n) Example: Binary Search 
public class BinarySearch {
    public static int binarySearch(int[] arr, int target) {
        int left = 0;
        int right = arr.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            
            if (arr[mid] == target) {
                return mid;
            } else if (arr[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        return -1;
    }
    
    public static void main(String[] args) {
        int[] numbers = {1, 3, 5, 7, 9, 11, 13, 15};
        int target = 9;
        int result = binarySearch(numbers, target);
        
        if (result != -1) {
            System.out.println("Element found at index: " + result);
        } else {
            System.out.println("Element not found");
        }
    }
}
Java O(log n) Example: Euclidean Algorithm for GCD 
public class EuclideanAlgorithm {
    public static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
    
    public static void main(String[] args) {
        System.out.println("GCD of 48 and 18 is: " + gcd(48, 18));
    }
}

Execution Time Calculation for GCD Algorithm

For GCD(48,18) on a MacBook Pro M4 (4.4 GHz CPU):

\[ \text{Cycle Time} = \frac{1}{4.4 \times 10^9} = 2.27 \times 10^{-10} \text{ seconds} \]

\[ \text{Total CPU Cycles} = 3 \text{ iterations} \times 15 \text{ cycles/iteration} = 45 \text{ cycles} \]

\[ \text{Execution Time} = 45 \times 2.27 \times 10^{-10} = 1.02 \times 10^{-8} \text{ seconds} = 10.2 \text{ nanoseconds} \]

O(n) - Linear Time

The runtime increases linearly with the size of the input.

Examples of O(n) Operations

  • Finding the maximum or minimum element in an unsorted array
  • Checking if an element exists in an unsorted array
  • Calculating the sum of all elements in an array
  • Traversing a linked list
Java O(n) Example: Finding Maximum Value 
public class LinearTimeExample {
    public static int findMax(int[] arr) {
        if (arr.length == 0) {
            throw new IllegalArgumentException("Array cannot be empty");
        }
        
        int max = arr[0];
        
        // Iterate through each element - O(n) operation
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > max) {
                max = arr[i];
            }
        }
        
        return max;
    }
    
    public static void main(String[] args) {
        int[] numbers = {3, 1, 9, 7, 5, 11, 2, 8, 6, 4};
        int maxValue = findMax(numbers);
        System.out.println("Maximum value: " + maxValue);
    }
}

O(n log n) - Linearithmic Time

The runtime increases in a linearithmic manner, combining linear and logarithmic growth.

Examples of O(n log n) Operations

  • Efficient sorting algorithms like Merge Sort, Quick Sort (average case), and Heap Sort
  • Constructing a binary search tree from a sorted array
  • Some divide and conquer algorithms
Java O(n log n) Example: Merge Sort 
public class MergeSort {
    public static void mergeSort(int[] arr) {
        if (arr.length <= 1) {
            return;
        }
        
        // Divide the array into two halves
        int mid = arr.length / 2;
        int[] left = new int[mid];
        int[] right = new int[arr.length - mid];
        
        System.arraycopy(arr, 0, left, 0, mid);
        System.arraycopy(arr, mid, right, 0, arr.length - mid);
        
        // Recursively sort both halves
        mergeSort(left);
        mergeSort(right);
        
        // Merge the sorted halves
        merge(arr, left, right);
    }
    
    private static void merge(int[] arr, int[] left, int[] right) {
        int i = 0, j = 0, k = 0;
        
        while (i < left.length && j < right.length) {
            if (left[i] <= right[j]) {
                arr[k++] = left[i++];
            } else {
                arr[k++] = right[j++];
            }
        }
        
        while (i < left.length) {
            arr[k++] = left[i++];
        }
        
        while (j < right.length) {
            arr[k++] = right[j++];
        }
    }
    
    public static void main(String[] args) {
        int[] numbers = {3, 1, 9, 7, 5, 11, 2, 8, 6, 4};
        mergeSort(numbers);
        System.out.print("Sorted array: ");
        for (int num : numbers) {
            System.out.print(num + " ");
        }
    }
}

O(n²) - Quadratic Time

The runtime increases quadratically with the size of the input.

Examples of O(n²) Operations

  • Simple sorting algorithms like Bubble Sort, Selection Sort, and Insertion Sort
  • Checking all pairs of elements in an array
  • Some matrix operations
Java O(n²) Example: Bubble Sort 
public class BubbleSort {
    public static void bubbleSort(int[] arr) {
        int n = arr.length;
        boolean swapped;
        
        for (int i = 0; i < n - 1; i++) {
            swapped = false;
            
            for (int j = 0; j < n - i - 1; j++) {
                if (arr[j] > arr[j + 1]) {
                    // Swap elements
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    swapped = true;
                }
            }
            
            // If no two elements were swapped, array is sorted
            if (!swapped) {
                break;
            }
        }
    }
    
    public static void main(String[] args) {
        int[] numbers = {3, 1, 9, 7, 5, 11, 2, 8, 6, 4};
        bubbleSort(numbers);
        System.out.print("Sorted array: ");
        for (int num : numbers) {
            System.out.print(num + " ");
        }
    }
}

Total Number of Iterations in Bubble Sort

Bubble Sort operates with two nested loops:

  • Outer Loop (i): Runs n times
  • Inner Loop (j): Runs (n - i - 1) times

The total number of comparisons is calculated using the formula:

\[ \sum_{i=0}^{n-1} (n-i-1) = \frac{n(n-1)}{2} \]

For n = 10: \[ \frac{10 \times 9}{2} = 45 \text{ comparisons} \]

O(2ⁿ) - Exponential Time

The runtime doubles with each additional input element. These algorithms are very inefficient for large inputs.

Examples of O(2ⁿ) Operations

  • Recursive Fibonacci without memoization
  • Solving the traveling salesman problem with brute force
  • Generating all subsets of a set
Java O(2ⁿ) Example: Recursive Fibonacci 
public class Fibonacci {
    // Inefficient recursive Fibonacci - O(2^n) time complexity
    public static int fibonacci(int n) {
        if (n <= 1) {
            return n;
        }
        return fibonacci(n - 1) + fibonacci(n - 2);
    }
    
    // Optimized Fibonacci with memoization - O(n) time complexity
    public static int fibonacciMemo(int n) {
        return fibonacciMemoHelper(n, new int[n + 1]);
    }
    
    private static int fibonacciMemoHelper(int n, int[] memo) {
        if (n <= 1) {
            return n;
        }
        if (memo[n] != 0) {
            return memo[n];
        }
        memo[n] = fibonacciMemoHelper(n - 1, memo) + fibonacciMemoHelper(n - 2, memo);
        return memo[n];
    }
    
    public static void main(String[] args) {
        int n = 10;
        System.out.println("Fibonacci(" + n + ") = " + fibonacci(n));
        System.out.println("Fibonacci with memoization(" + n + ") = " + fibonacciMemo(n));
    }
}
