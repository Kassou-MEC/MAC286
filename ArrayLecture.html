Array Basics üì¶

Array: A fixed-size, indexed sequence of values of the same type stored contiguously in memory.
Fixed size: Once created, an array‚Äôs length cannot change. Need resize? Consider ArrayList<T>.

Declaring & Initializing

Common ways to declare and initialize integer arrays:

Java 
// Method 1: Declare, then create with a fixed length (defaults to 0s)
int[] numbers;
numbers = new int[5];              // [0, 0, 0, 0, 0]

// Method 2: Declare + create in one line
int[] scores = new int[10];        // length = 10

// Method 3: Create with literal values (length inferred)
int[] primes = { 2, 3, 5, 7, 11 }; // length = 5

// Method 4: Alternative "new int[]" literal form
int[] ages = new int[]{ 21, 18, 25, 30 };
  • Default values: numeric types ‚Üí 0, boolean ‚Üí false, object refs ‚Üí null.
  • Length: available as arr.length (a field, not a method).
  • Bounds: indices run 0..length-1. Accessing outside throws ArrayIndexOutOfBoundsException.

Other Types

Java 
// Strings, doubles, booleans, chars
String[] names = { "Alice", "Bob", "Charlie" };
double[] prices = new double[100];     // default 0.0
boolean[] flags = new boolean[8];      // default false
char[] letters = "Hello".toCharArray();// ['H','e','l','l','o']

String vs char[]: String is an immutable object; toCharArray() gives a mutable array of characters.

2D & Jagged Arrays

A 2D array is an array of arrays (matrix). A jagged array has rows with different lengths.

Java 
// 2D array (3 rows √ó 4 cols)
int[][] matrix = new int[3][4];

// With values
int[][] grid = {
    { 1, 2, 3 },
    { 4, 5, 6 },
    { 7, 8, 9 }
};

// Jagged (rows with different lengths)
int[][] jagged = new int[3][];
jagged[0] = new int[5];
jagged[1] = new int[3];
jagged[2] = new int[4];

Memory note: A 2D Java array isn‚Äôt a true matrix; each row is its own array (hence ‚Äújagged‚Äù rows are allowed).

Core Array Operations ‚öôÔ∏è

Random access (O(1)): arr[i] is constant-time because arrays are contiguous in memory.

Access, Update, Loop

Java 
int[] arr = { 10, 20, 30, 40, 50 };

// Access
int first = arr[0];                 // 10
int last  = arr[arr.length - 1];    // 50

// Update
arr[2] = 35;                        // {10, 20, 35, 40, 50}

// Loop (classic for)
for (int i = 0; i < arr.length; i++) {
    System.out.println("Index " + i + ": " + arr[i]);
}

// Enhanced for
for (int x : arr) {
    System.out.println(x);
}
  • Classic for: use when you need the index i.
  • Enhanced for: simple value iteration; you can‚Äôt modify the array‚Äôs structure (still fine to assign via index).
  • Complexity: both loops are O(n) time, O(1) extra space.

Useful Utilities

Java 
public final class ArrayUtils {

    // O(n): find max
    public static int max(int[] arr) {
        int best = arr[0];
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > best) best = arr[i];
        }
        return best;
    }

    // O(n): sum
    public static int sum(int[] arr) {
        int total = 0;
        for (int x : arr) total += x;
        return total;
    }

    // O(n): reverse in place (two pointers)
    public static void reverse(int[] arr) {
        int L = 0, R = arr.length - 1;
        while (L < R) {
            int tmp = arr[L];
            arr[L] = arr[R];
            arr[R] = tmp;
            L++; R--;
        }
    }
}
  • max: scans once; track best so far.
  • sum: add as you go; beware of integer overflow for very large sums.
  • reverse: swap symmetric elements from ends inward (two-pointer technique).

Arrays & OOP üé≠

Array of objects: Stores references to objects. You must instantiate each element before use.

Arrays of Custom Objects

Java 
// Simple Student class
class Student {
    private String name;
    private int grade;

    public Student(String name, int grade) {
        this.name  = name;
        this.grade = grade;
    }

    public String getName()  { return name;  }
    public int    getGrade() { return grade; }
}

// Creating and initializing an object array
Student[] classroom = new Student[3]; // {null, null, null}
classroom[0] = new Student("Alice", 95);
classroom[1] = new Student("Bob",   87);
classroom[2] = new Student("Cara",  92);

Nulls at start: new Student[3] creates storage for 3 references, all null until you assign real objects.

Managing Object Arrays (Encapsulation)

Java 
final class Library {
    private Book[] books;
    private int count;

    public Library(int capacity) {
        books = new Book[capacity];
        count = 0;
    }

    // O(1) add (if capacity allows)
    public boolean add(Book b) {
        if (count == books.length) return false; // full
        books[count++] = b;
        return true;
    }

    // O(n): linear search by title
    public Book find(String title) {
        for (int i = 0; i < count; i++) {
            if (books[i].getTitle().equals(title)) return books[i];
        }
        return null;
    }

    // O(n): remove and left-shift the tail
    public boolean remove(String title) {
        for (int i = 0; i < count; i++) {
            if (books[i].getTitle().equals(title)) {
                for (int j = i; j < count - 1; j++) books[j] = books[j + 1];
                books[--count] = null; // avoid stale reference
                return true;
            }
        }
        return false;
    }
}

final class Book {
    private final String title;
    private final String author;
    private final int    year;

    public Book(String t, String a, int y) {
        title = t; author = a; year = y;
    }
    public String getTitle()  { return title;  }
    public String getAuthor() { return author; }
    public int    getYear()   { return year;   }
}
  • Encapsulation: fields are private; behavior exposed via methods.
  • Remove: shifts elements so remaining books stay contiguous; last slot nulled to prevent memory leak.

Polymorphism with Arrays

Java 
abstract class Shape {
    protected String color;
    public Shape(String color) { this.color = color; }
    public abstract double area();
    public abstract double perimeter();
}

final class Circle extends Shape {
    private final double r;
    public Circle(String color, double r) {
        super(color); this.r = r;
    }
    public double area()      { return Math.PI * r * r; }
    public double perimeter() { return 2 * Math.PI * r; }
}

final class Rectangle extends Shape {
    private final double w, h;
    public Rectangle(String color, double w, double h) {
        super(color); this.w = w; this.h = h;
    }
    public double area()      { return w * h; }
    public double perimeter() { return 2 * (w + h); }
}

// Polymorphic array
Shape[] shapes = {
    new Circle("Red", 5.0),
    new Rectangle("Blue", 4.0, 6.0),
    new Circle("Green", 3.0),
};
double totalArea = 0;
for (Shape s : shapes) totalArea += s.area();

Polymorphism: the area() call dispatches to the appropriate subclass implementation at runtime.

Ordered (Sorted) Arrays üìä

Definition: Elements arranged in non-decreasing (or non-increasing) order. Enables binary search in O(log n).

Built-in Sorting

Java 
import java.util.Arrays;
import java.util.Collections;

int[] a = { 64, 34, 25, 12, 22, 11, 90 };
Arrays.sort(a); // ascending

Integer[] b = { 5, 2, 8, 1, 9 };
Arrays.sort(b, Collections.reverseOrder()); // descending (needs wrapper type)

Why Integer[] for reverse? Java‚Äôs reverseOrder() comparator works with objects, not primitives.

Binary Search (Iterative)

Java 
// Returns index of target or -1 if not found (array must be sorted)
public static int binarySearch(int[] arr, int target) {
    int L = 0, R = arr.length - 1;
    while (L <= R) {
        int mid = L + (R - L) / 2;   // avoids (L+R) overflow
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) L = mid + 1;
        else R = mid - 1;
    }
    return -1;
}
  • Invariant: search range is [L, R] inclusive; shrink toward the target.
  • Complexity: O(log n) time, O(1) extra space.

First/Last Occurrence (Duplicates)

Java 
public static int firstIndex(int[] arr, int target) {
    int L = 0, R = arr.length - 1, ans = -1;
    while (L <= R) {
        int mid = L + (R - L) / 2;
        if (arr[mid] >= target) R = mid - 1;
        else L = mid + 1;
        if (arr[mid] == target) ans = mid; // keep going left
    }
    return ans;
}

public static int lastIndex(int[] arr, int target) {
    int L = 0, R = arr.length - 1, ans = -1;
    while (L <= R) {
        int mid = L + (R - L) / 2;
        if (arr[mid] <= target) L = mid + 1;
        else R = mid - 1;
        if (arr[mid] == target) ans = mid; // keep going right
    }
    return ans;
}

Trick: when you find the target, don‚Äôt stop‚Äîshift the search to the side you need and remember the best index so far.

Sorting Algorithms üîÑ

Stable sort: preserves relative order of equal keys (e.g., Merge Sort).
In-place: uses O(1) extra space (ignoring recursion stack), e.g., Insertion Sort, Quick Sort.

Implementations

Bubble Sort (early exit)

Java 
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int t = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = t;
                swapped = true;
            }
        }
        if (!swapped) break; // already sorted
    }
}

Repeatedly ‚Äúbubble‚Äù the largest element to the end. Early exit stops once a pass makes no swaps.

Insertion Sort

Java 
public static void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

Builds the sorted portion one element at a time; fast for nearly sorted inputs, stable, in-place.

Quick Sort

Java 
public static void quickSort(int[] a, int low, int high) {
    if (low < high) {
        int p = partition(a, low, high);
        quickSort(a, low, p - 1);
        quickSort(a, p + 1, high);
    }
}

private static int partition(int[] a, int low, int high) {
    int pivot = a[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (a[j] < pivot) {
            i++;
            int t = a[i]; a[i] = a[j]; a[j] = t;
        }
    }
    int t = a[i + 1]; a[i + 1] = a[high]; a[high] = t;
    return i + 1;
}
  • Idea: partition into < pivot and ‚â• pivot, then recurse.
  • Average: O(n log n). Worst: O(n¬≤) (e.g., already sorted with bad pivot choice).
  • Space: O(log n) recursion depth on average.

Practice Problems üí™

Two Sum

Task: Given nums and target, return indices of two numbers adding to target. Idea: Hash map of value ‚Üí index.
Java 
import java.util.*;

public static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> pos = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (pos.containsKey(need)) return new int[]{ pos.get(need), i };
        pos.put(nums[i], i);
    }
    return new int[0];
}
  • Map: stores numbers we‚Äôve seen and their positions.
  • Check: for each nums[i], look up target - nums[i].
  • Complexity: O(n) time, O(n) space.

Move Zeros

Task: Move all zeros to the end, keep order of non-zeros. Idea: Two-pointer compaction.
Java 
public static void moveZeros(int[] nums) {
    int k = 0;                       // next position for a non-zero
    for (int x : nums) if (x != 0) nums[k++] = x;
    while (k < nums.length) nums[k++] = 0;
}

First pass copies non-zeros forward; second pass pads the rest with 0s. O(n) time, O(1) extra space.

Missing Number (0..n)

Task: Given n distinct numbers in [0..n], find the missing one. Ideas: Sum formula or XOR trick.
Java 
public static int missingNumberSum(int[] nums) {
    int n = nums.length;
    int expected = n * (n + 1) / 2;
    int actual = 0;
    for (int x : nums) actual += x;
    return expected - actual;
}

public static int missingNumberXor(int[] nums) {
    int x = nums.length;
    for (int i = 0; i < nums.length; i++) x ^= i ^ nums[i];
    return x;
}
  • Sum: arithmetic series minus actual sum ‚Üí missing number.
  • XOR: each index & value cancels; leftover is the missing value. O(n) time, O(1) space.

Advanced Challenges üèÜ

Product of Array Except Self (No Division, O(n))

Idea: Prefix products √ó suffix products. Space O(1) if you reuse the output array.
Java 
public static int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] res = new int[n];

    // left products
    res[0] = 1;
    for (int i = 1; i < n; i++) res[i] = res[i - 1] * nums[i - 1];

    // right product accumulator
    int right = 1;
    for (int i = n - 1; i >= 0; i--) {
        res[i] *= right;
        right *= nums[i];
    }
    return res;
}

For each index, multiply the product of all items to its left by the product of all items to its right.

3Sum (All Unique Triplets Summing to 0)

Idea: Sort + two pointers; skip duplicates carefully to avoid repeated triplets.
Java 
import java.util.*;

public static List<List<Integer>> threeSum(int[] nums) {
    List<List<Integer>> out = new ArrayList<>();
    Arrays.sort(nums);
    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue; // skip dup anchors
        int L = i + 1, R = nums.length - 1;
        while (L < R) {
            int s = nums[i] + nums[L] + nums[R];
            if (s == 0) {
                out.add(Arrays.asList(nums[i], nums[L], nums[R]));
                while (L < R && nums[L] == nums[L + 1]) L++;
                while (L < R && nums[R] == nums[R - 1]) R--;
                L++; R--;
            } else if (s < 0) L++;
            else R--;
        }
    }
    return out;
}

Why sort? Sorting enables the two-pointer sweep and simplifies duplicate handling. Complexity O(n¬≤).

Glossary üìö

  • Array: Fixed-size, indexed collection of same-type elements.
  • Index: Position of an element (0-based in Java).
  • Reference: Address pointing to an object on the heap.
  • Heap: Memory region where objects live.
  • Stack: Memory for call frames and local variables.
  • O(1): Constant time.
  • O(n): Linear time.
  • O(log n): Logarithmic time.
  • Stable sort: Keeps relative order of equal keys.
  • In-place: Uses O(1) extra working space (ignoring recursion).
  • Binary search: O(log n) search on sorted arrays.
  • Two pointers: Using two indices that move to find structure.
