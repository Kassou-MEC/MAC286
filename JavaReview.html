Variables and Data Types

Storing and manipulating data in Java

Variables are containers for storing data values. In Java, each variable must be declared with a specific data type.

Java is a statically-typed language, which means that all variables must be declared with a specific type before they can be used. This allows for compile-time type checking, which helps prevent many common programming errors.

Primitive Data Types

Java has 8 primitive data types that represent single values:

  • byte: 8-bit integer (-128 to 127)
  • short: 16-bit integer (-32,768 to 32,767)
  • int: 32-bit integer (most common)
  • long: 64-bit integer
  • float: 32-bit floating point
  • double: 64-bit floating point (most common)
  • boolean: true or false
  • char: single character

Reference Data Types

Java also has reference types, which refer to objects:

  • String: sequence of characters (immutable)
  • Arrays: collections of elements
  • Classes: custom data types

Variable Declaration Example

public class VariablesExample {
    public static void main(String[] args) {
        // Integer types
        int age = 25;

        System.out.println("Is Java fun? " + isJavaFun);

}

Explanation

This example demonstrates declaring variables of different data types in Java:

  • int and long for integer values. Note the 'L' suffix for long literals.
  • double and float for floating-point numbers. Note the 'f' suffix for float literals.
  • char for single characters, using single quotes.
  • boolean for true/false values.
  • String for text (note: String is a class, not a primitive type), using double quotes.

Each variable declaration follows the pattern: type variableName = value;. Java requires explicit type declaration, which helps catch errors at compile time rather than runtime.

Remember

Java is a statically-typed language, which means variables must be declared with a specific type before they can be used. This provides compile-time type checking, which helps prevent many common programming errors.

Conditionals and Loops

Controlling program flow in Java

Conditionals and loops allow you to control the flow of your program based on conditions and repeat operations. Conditionals enable decision-making in code, while loops allow for repeated execution of code blocks.

Control structures are fundamental to programming as they allow programs to respond differently to different inputs and situations, making programs dynamic and responsive.

Conditional Statements

  • if: Executes a block of code if a condition is true
  • if-else: Executes one block if true, another if false
  • if-else-if: Checks multiple conditions in sequence
  • switch: Selects one of many code blocks to execute based on a value

Loop Statements

  • for: loops through a block a specific number of times
  • while: loops while a condition remains true
  • do-while: similar to while but executes at least once
  • Enhanced for: iterates through arrays and collections

Conditionals Example

public class ConditionalsExample {
    public static void main(String[] args) {
        int number = 15;
        
        // If-else example
        if (number % 2 == 0) {
            System.out.println(number + " is even");
        } else {
            System.out.println(number + " is odd");
        }
        
        // If-else-if example
        if (number < 0) {
            System.out.println("Negative number");
        } else if (number > 0) {
            System.out.println("Positive number");
        } else {
            System.out.println("Zero");
        }
        
        // Switch example
        int day = 3;
        String dayName;
        
        switch (day) {
            case 1:
                dayName = "Monday";
                break;
            case 2:
                dayName = "Tuesday";
                break;
            case 3:
                dayName = "Wednesday";
                break;
            case 4:
                dayName = "Thursday";
                break;
            case 5:
                dayName = "Friday";
                break;
            case 6:
                dayName = "Saturday";
                break;
            case 7:
                dayName = "Sunday";
                break;
            default:
                dayName = "Invalid day";
        }
        System.out.println("Day " + day + " is " + dayName);
    }
}

Explanation

This example shows different conditional statements in Java:

  • if-else: Checks if a number is even or odd using the modulus operator (%). The modulus operator returns the remainder of a division operation.
  • if-else-if: Checks if a number is negative, positive, or zero. Conditions are evaluated in order, and only the first true condition executes.
  • switch: Selects a day name based on a numeric value. The switch statement compares the value against each case label. The break statement is crucial to prevent "fall-through" to subsequent cases.

Conditional statements allow programs to make decisions and execute different code paths based on conditions, making programs more flexible and responsive to different situations.

Loops Example

public class LoopsExample {
    public static void main(String[] args) {
        // For loop example
        System.out.println("For loop (1 to 5):");
        for (int i = 1; i <= 5; i++) {
            System.out.print(i + " ");
        }
        System.out.println();
        
        // While loop example
        System.out.println("While loop (5 to 1):");
        int j = 5;
        while (j >= 1) {
            System.out.print(j + " ");
            j--;
        }
        System.out.println();
        
        // Do-while loop example
        System.out.println("Do-while loop (1 to 3):");
        int k = 1;
        do {
            System.out.print(k + " ");
            k++;
        } while (k <= 3);
        System.out.println();
        
        // Nested loops example (multiplication table)
        System.out.println("Multiplication table (1-3):");
        for (int x = 1; x <= 3; x++) {
            for (int y = 1; y <= 3; y++) {
                System.out.print(x * y + "\t");
            }
            System.out.println();
        }
        
        // Break and continue example
        System.out.println("Break and continue example:");
        for (int i = 1; i <= 10; i++) {
            if (i == 5) {
                continue; // Skip the rest of the loop for i=5
            }
            if (i == 8) {
                break;  // Exit the loop when i=8
            }
            System.out.print(i + " ");
        }
    }
}

Explanation

This example demonstrates different loop structures in Java:

  • for loop: Counts from 1 to 5 with a fixed number of iterations. The for loop has three parts: initialization (int i = 1), condition (i <= 5), and increment (i++).
  • while loop: Counts down from 5 to 1, checking condition before each iteration. The loop continues as long as the condition remains true.
  • do-while loop: Counts from 1 to 3, executing at least once before checking condition. Unlike the while loop, the condition is checked after the loop body executes.
  • nested loops: Creates a multiplication table using a for loop inside another for loop. The inner loop completes all its iterations for each iteration of the outer loop.
  • break and continue: The break statement exits the loop entirely, while continue skips the current iteration and moves to the next one.

Loops are essential for performing repetitive tasks efficiently. The choice of loop type depends on whether you know the number of iterations in advance and whether the loop should execute at least once.

Methods

Organizing code into reusable blocks

Methods are blocks of code that perform a specific task and can be reused throughout your program. They help with code organization, reusability, and maintenance.

Methods allow you to break down complex problems into smaller, manageable pieces. They promote code reuse, reduce duplication, and make programs easier to read, debug, and maintain. Methods can take parameters as input and can return a value as output.

Method Components

  • Access modifier: public, private, protected, or package-private (default)
  • Return type: data type of the value returned, or void if no return value
  • Method name: should be a verb describing the action
  • Parameters: input values the method needs to work
  • Method body: the code that performs the task

Method Overloading

Java allows method overloading, which means multiple methods can have the same name but different parameters. This is useful when you want to perform similar operations with different types or numbers of inputs.

Methods Example

public class MethodsExample {
    
    // Method with no parameters and no return value
    public static void greet() {
        System.out.println("Hello, welcome to Java programming!");
    }
    
    // Method with parameters and no return value
    public static void greetUser(String name) {
        System.out.println("Hello, " + name + "! Welcome to Java programming!");
    }
    
    // Method with parameters and return value
    public static int addNumbers(int a, int b) {
        return a + b;
    }
    
    // Method overloading - same name, different parameters
    public static double addNumbers(double a, double b) {
        return a + b;
    }
    
    // Method with multiple parameters
    public static void printUserInfo(String name, int age, boolean isStudent) {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Student: " + isStudent);
    }
    
    // Method calling another method
    public static void processUser(String name, int age) {
        greetUser(name);
        System.out.println("You are " + age + " years old");
    }
    
    public static void main(String[] args) {
        // Calling methods
        greet();
        greetUser("Alice");
        
        int sum1 = addNumbers(5, 7);
        System.out.println("Sum of integers: " + sum1);
        
        double sum2 = addNumbers(3.5, 2.8);
        System.out.println("Sum of doubles: " + sum2);
        
        // Using a method in a calculation
        int result = addNumbers(10, 20) * 2;
        System.out.println("Result: " + result);
        
        // Method with multiple parameters
        printUserInfo("Bob", 25, true);
        
        // Method calling another method
        processUser("Charlie", 30);
    }
}

Explanation

This example demonstrates different aspects of methods in Java:

  • Simple method: greet() with no parameters and no return value (void). This method performs an action but doesn't return any data.
  • Parameterized method: greetUser(String name) with a parameter but no return value. Parameters allow methods to accept input values.
  • Method with return value: addNumbers(int a, int b) that returns the sum of two integers. The return statement sends a value back to the caller.
  • Method overloading: Two addNumbers methods with different parameter types (int vs double). Java determines which method to call based on the argument types.
  • Multiple parameters: printUserInfo() accepts three parameters of different types.
  • Method calling another method: processUser() calls greetUser() and performs additional operations.

Method overloading allows using the same method name for different parameter types, making the code more intuitive and reducing the need for different method names for similar operations.

Remember

The static keyword means the method belongs to the class itself rather than to instances of the class. This allows calling the method without creating an object. Static methods can only directly call other static methods and access static variables.

Arrays and Strings

Working with collections of data and text

Arrays are containers that hold a fixed number of values of a single type. Strings are objects that represent sequences of characters.

Arrays provide a way to store and access multiple values of the same type using a single variable name and an index. Strings are immutable sequences of characters that provide numerous methods for manipulation and examination of text data.

Array Characteristics

  • Fixed size once created
  • Can hold primitives or objects
  • Zero-based indexing (first element is at index 0)
  • Elements are stored in contiguous memory locations
  • Array length is available via the length property

String Characteristics

  • Immutable - cannot be changed after creation
  • String literals are stored in a string pool
  • Many useful methods for manipulation
  • Can be created with string literals or with new keyword
  • Implements the CharSequence interface

Arrays Example

public class ArraysExample {
    public static void main(String[] args) {
        // Array declaration and initialization
        int[] numbers = {10, 20, 30, 40, 50};
        
        // Accessing array elements
        System.out.println("First element: " + numbers[0]);
        System.out.println("Third element: " + numbers[2]);
        
        // Modifying array elements
        numbers[1] = 25;
        System.out.println("Modified second element: " + numbers[1]);
        
        // Array length
        System.out.println("Array length: " + numbers.length);
        
        // Iterating through array with for loop
        System.out.print("All elements: ");
        for (int i = 0; i < numbers.length; i++) {
            System.out.print(numbers[i] + " ");
        }
        System.out.println();
        
        // Enhanced for loop (for-each)
        System.out.print("Using for-each: ");
        for (int num : numbers) {
            System.out.print(num + " ");
        }
        System.out.println();
        
        // Array of strings
        String[] names = {"Alice", "Bob", "Charlie"};
        for (String name : names) {
            System.out.println("Name: " + name);
        }
        
        // Multi-dimensional array
        int[][] matrix = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };
        
        System.out.println("Matrix element at [1][2]: " + matrix[1][2]);
        
        // Iterating through multi-dimensional array
        System.out.println("Matrix elements:");
        for (int i = 0; i < matrix.length; i++) {
            for (int j = 0; j < matrix[i].length; j++) {
                System.out.print(matrix[i][j] + " ");
            }
            System.out.println();
        }
    }
}

Explanation

This example demonstrates working with arrays in Java:

  • Declaration and initialization: Creating an array with initial values using curly braces {}
  • Accessing elements: Using index notation (e.g., numbers[0]). Array indices start at 0.
  • Modifying elements: Changing values at specific indices using assignment
  • Array length: Using the length property (not a method call)
  • Iteration: Using both traditional for loops and enhanced for-each loops. The for-each loop is simpler but doesn't provide access to the index.
  • String arrays: Creating and working with arrays of strings
  • Multi-dimensional arrays: Arrays of arrays, useful for representing matrices or grids

Arrays are fundamental data structures that store elements of the same type in contiguous memory locations. They provide fast access to elements by index but have a fixed size once created.

Strings Example

public class StringsExample {
    public static void main(String[] args) {
        // Different ways to create strings
        String str1 = "Hello";  // String literal (stored in string pool)
        String str2 = new String("World");  // Using constructor (new object)
        
        // String concatenation
        String message = str1 + " " + str2;
        System.out.println(message);
        
        // String length
        System.out.println("Length: " + message.length());
        
        // Character at specific position
        System.out.println("Character at index 1: " + message.charAt(1));
        
        // Substring
        System.out.println("Substring (0-5): " + message.substring(0, 5));
        
        // String comparison
        String s1 = "Java";
        String s2 = "Java";
        String s3 = new String("Java");
        String s4 = "java";
        
        System.out.println("s1 == s2: " + (s1 == s2));  // true (same object in string pool)
        System.out.println("s1 == s3: " + (s1 == s3));  // false (different objects)
        System.out.println("s1.equals(s3): " + s1.equals(s3));  // true (same content)
        System.out.println("s1.equals(s4): " + s1.equals(s4));  // false (case difference)
        System.out.println("s1.equalsIgnoreCase(s4): " + s1.equalsIgnoreCase(s4));  // true
        
        // String methods
        String text = "  Java Programming  ";
        System.out.println("Original: '" + text + "'");
        System.out.println("Trimmed: '" + text.trim() + "'");
        System.out.println("Uppercase: '" + text.toUpperCase() + "'");
        System.out.println("Lowercase: '" + text.toLowerCase() + "'");
        System.out.println("Contains 'Pro': " + text.contains("Pro"));
        System.out.println("Replaced: '" + text.replace("Java", "Python") + "'");
        System.out.println("Starts with 'Java': " + text.trim().startsWith("Java"));
        System.out.println("Ends with 'ing': " + text.trim().endsWith("ing"));
        
        // String splitting
        String csv = "Apple,Banana,Orange,Mango";
        String[] fruits = csv.split(",");
        System.out.println("Fruits: ");
        for (String fruit : fruits) {
            System.out.println("- " + fruit);
        }
        
        // String building efficiently
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 5; i++) {
            sb.append("Line ").append(i).append("\n");
        }
        System.out.println("StringBuilder result:\n" + sb.toString());
    }
}

Explanation

This example demonstrates working with strings in Java:

  • String creation: Using literals (pooled) vs. new keyword (new object). String literals are stored in a string pool to save memory.
  • Concatenation: Using the + operator to combine strings. For multiple concatenations, StringBuilder is more efficient.
  • String methods: length(), charAt(), substring(), etc. Strings provide numerous methods for examination and manipulation.
  • Comparison: Difference between == (reference equality) and equals() (content equality). equals() compares content, while == compares object references.
  • Useful methods: trim(), toUpperCase(), toLowerCase(), contains(), replace(), startsWith(), endsWith()
  • Splitting: Using split() to break a string into parts based on a delimiter
  • StringBuilder: For efficient string manipulation, especially in loops. Unlike Strings, StringBuilder is mutable.

Strings are immutable in Java, meaning any operation that modifies a string actually creates a new string object. This immutability provides thread safety and security benefits but can be inefficient for frequent modifications.

OOP Basics

Object-Oriented Programming Fundamentals

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data and code to manipulate that data.

OOP models software as a collection of objects that interact with each other. Each object is an instance of a class, which defines the object's properties (attributes) and behaviors (methods). OOP helps manage complexity in large software systems by providing principles like encapsulation, inheritance, and polymorphism.

OOP Principles

  • Encapsulation: Bundling data and methods that operate on that data, and restricting direct access to some of an object's components. This is achieved through access modifiers (public, private, protected) and helps maintain data integrity.
  • Inheritance: Creating new classes based on existing ones, allowing code reuse and establishing relationships between classes. A subclass inherits fields and methods from its superclass.
  • Polymorphism: Ability to present the same interface for different underlying forms. This allows objects of different classes to be treated as objects of a common superclass.
  • Abstraction: Hiding complex implementation details and showing only essential features. Abstract classes and interfaces are used to define abstractions in Java.

Class Components

  • Fields: Variables that hold the state of an object (also called attributes or properties)
  • Constructors: Special methods to initialize new objects. They have the same name as the class and no return type.
  • Methods: Functions that define object behavior and operations
  • Access modifiers: public, private, protected controls visibility of class members

Class Example

// A simple class representing a BankAccount
public class BankAccount {
    // Fields (attributes) - encapsulated with private access
    private String accountNumber;
    private String accountHolder;
    private double balance;
    
    // Constructor - initializes new objects
    public BankAccount(String accountNumber, String accountHolder, double initialBalance) {
        this.accountNumber = accountNumber;
        this.accountHolder = accountHolder;
        this.balance = initialBalance;
    }
    
    // Default constructor (overloading)
    public BankAccount() {
        this("000000", "Unknown", 0.0);
    }
    
    // Methods (behaviors)
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: $" + amount);
        } else {
            System.out.println("Invalid deposit amount");
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrew: $" + amount);
        } else {
            System.out.println("Invalid withdrawal amount or insufficient funds");
        }
    }
    
    // Accessor (getter) methods - provide controlled access to private fields
    public String getAccountNumber() {
        return accountNumber;
    }
    
    public String getAccountHolder() {
        return accountHolder;
    }
    
    public double getBalance() {
        return balance;
    }
    
    // Mutator (setter) method - provide controlled modification of private fields
    public void setAccountHolder(String accountHolder) {
        if (accountHolder != null && !accountHolder.trim().isEmpty()) {
            this.accountHolder = accountHolder;
        } else {
            System.out.println("Invalid account holder name");
        }
    }
    
    // toString method to represent object as string
    public String toString() {
        return "BankAccount[accountNumber=" + accountNumber + 
               ", accountHolder=" + accountHolder + 
               ", balance=$" + balance + "]";
    }
}

// Main class to test BankAccount
public class BankAccountTest {
    public static void main(String[] args) {
        // Create a BankAccount object
        BankAccount account = new BankAccount("123456", "John Doe", 1000.0);
        
        // Use object methods
        System.out.println(account);
        account.deposit(500.0);
        account.withdraw(200.0);
        System.out.println("Current balance: $" + account.getBalance());
        
        // Change account holder name
        account.setAccountHolder("John Smith");
        System.out.println("New account holder: " + account.getAccountHolder());
        
        // Try to set invalid name
        account.setAccountHolder("");
        
        // Create another account using default constructor
        BankAccount defaultAccount = new BankAccount();
        System.out.println(defaultAccount);
        
        // Demonstrate encapsulation - this would cause a compilation error:
        // account.balance = 1000000; // Error: balance has private access
    }
}

Explanation

This example demonstrates Object-Oriented Programming basics in Java:

  • Class definition: BankAccount class with fields, constructors, and methods. A class is a blueprint for creating objects.
  • Encapsulation: Using private fields with public getters and setters. This protects the object's internal state from invalid modifications.
  • Constructors: Parameterized constructor and default constructor (overloading). Constructors initialize new objects.
  • Methods: deposit(), withdraw(), and accessor/mutator methods. Methods define object behavior.
  • this keyword: Referring to the current object instance. Used to disambiguate between instance variables and parameters.
  • Object creation: Using the new keyword to create instances. Each object has its own copy of instance variables.
  • toString() method: Overriding to provide a string representation of the object. This is called automatically when an object is printed.

OOP allows modeling real-world entities as objects with both data and behavior, promoting code organization and reusability. Encapsulation ensures that an object's internal state remains consistent by controlling how it can be modified.

Remember

Access modifiers control visibility:

  • private: Accessible only within the class. Provides the highest level of encapsulation.
  • protected: Accessible within the package and subclasses. Useful for inheritance hierarchies.
  • public: Accessible from anywhere. Used for methods that form the class's public interface.
  • default (no modifier): Accessible within the package. Provides package-level encapsulation.

Inheritance and Polymorphism

Extending classes and implementing flexibility

Inheritance allows a class to inherit properties and behaviors from another class. Polymorphism allows objects of different classes to be treated as objects of a common superclass.

Inheritance creates an "is-a" relationship between classes, where a subclass is a specialized version of its superclass. Polymorphism enables writing more flexible and reusable code by allowing methods to work with objects of various types through a common interface.

Inheritance Concepts

  • Superclass (parent): The class being inherited from. Also called base class.
  • Subclass (child): The class that inherits from another class. Also called derived class.
  • extends keyword: Used to create a subclass. Java supports single inheritance for classes.
  • super keyword: Refers to the superclass constructor or methods. Used to call superclass constructors or overridden methods.
  • Method overriding: Subclass provides a specific implementation of a method defined in its superclass.

Polymorphism Concepts

  • Dynamic method dispatch: Method implementation determined at runtime based on the actual object type.
  • Upcasting: Treating a subclass object as superclass object. Always safe and implicit.
  • Downcasting: Treating a superclass reference as subclass object. Requires explicit casting and can cause ClassCastException.
  • Abstract classes: Classes that cannot be instantiated, used as base classes for inheritance.
  • Interfaces: Define contracts that classes can implement, enabling multiple inheritance of types.

Inheritance Example

// Superclass
class Vehicle {
    protected String brand;
    protected int year;
    
    public Vehicle(String brand, int year) {
        this.brand = brand;
        this.year = year;
    }
    
    public void start() {
        System.out.println("The vehicle is starting");
    }
    
    public void stop() {
        System.out.println("The vehicle is stopping");
    }
    
    public String getInfo() {
        return "Brand: " + brand + ", Year: " + year;
    }
}

// Subclass 1
class Car extends Vehicle {
    private int numberOfDoors;
    
    public Car(String brand, int year, int numberOfDoors) {
        super(brand, year);  // Call superclass constructor
        this.numberOfDoors = numberOfDoors;
    }
    
    // Method overriding - providing specific implementation
    public void start() {
        System.out.println("The car is starting with a key");
    }
    
    // Additional method specific to Car
    public void honk() {
        System.out.println("Beep beep!");
    }
    
    // Override getInfo method
    public String getInfo() {
        return super.getInfo() + ", Doors: " + numberOfDoors;
    }
}

// Subclass 2
class Motorcycle extends Vehicle {
    private boolean hasSidecar;
    
    public Motorcycle(String brand, int year, boolean hasSidecar) {
        super(brand, year);
        this.hasSidecar = hasSidecar;
    }
    
    // Method overriding
    public void start() {
        System.out.println("The motorcycle is starting with a kick");
    }
    
    // Override getInfo method
    public String getInfo() {
        return super.getInfo() + ", Has sidecar: " + hasSidecar;
    }
}

// Main class to test inheritance
public class InheritanceTest {
    public static void main(String[] args) {
        // Create objects
        Car car = new Car("Toyota", 2020, 4);
        Motorcycle motorcycle = new Motorcycle("Harley-Davidson", 2019, false);
        
        // Call methods
        System.out.println(car.getInfo());
        car.start();
        car.honk();  // Specific to Car
        car.stop();
        
        System.out.println();
        
        System.out.println(motorcycle.getInfo());
        motorcycle.start();
        motorcycle.stop();
        
        // Polymorphism example
        System.out.println("\n--- Polymorphism Example ---");
        Vehicle[] vehicles = new Vehicle[2];
        vehicles[0] = car;        // Upcasting: Car -> Vehicle
        vehicles[1] = motorcycle; // Upcasting: Motorcycle -> Vehicle
        
        for (Vehicle vehicle : vehicles) {
            vehicle.start();  // Dynamic method binding - calls appropriate version
            System.out.println(vehicle.getInfo());  // Also polymorphic
            System.out.println();
        }
        
        // Demonstrating downcasting
        System.out.println("--- Downcasting Example ---");
        for (Vehicle vehicle : vehicles) {
            if (vehicle instanceof Car) {
                Car carObject = (Car) vehicle; // Downcasting: Vehicle -> Car
                carObject.honk(); // Can now call Car-specific method
            }
        }
    }
}

Explanation

This example demonstrates inheritance and polymorphism in Java:

  • Inheritance: Car and Motorcycle classes extend the Vehicle class using the extends keyword. This creates an "is-a" relationship (Car is a Vehicle).
  • super keyword: Used to call superclass constructor and methods. The super() call must be the first statement in a constructor.
  • Method overriding: Subclasses provide their own implementation of start() and getInfo() methods. The @Override annotation can be used to explicitly declare overriding.
  • Polymorphism: Treating Car and Motorcycle objects as Vehicle objects in an array. This allows writing code that works with general types rather than specific ones.
  • Dynamic binding: The JVM determines which version of start() to call at runtime based on the actual object type. This enables polymorphic behavior.
  • Upcasting: Assigning subclass objects to superclass references (implicit and always safe).
  • Downcasting: Assigning superclass references to subclass references (explicit and requires type checking with instanceof).

Inheritance promotes code reuse, and polymorphism allows writing more flexible and maintainable code. Together, they form the foundation of object-oriented programming in Java.

Remember

Java supports single inheritance for classes (a class can extend only one superclass) but multiple inheritance for interfaces. Use the instanceof operator to check an object's type before downcasting to avoid ClassCastException.
