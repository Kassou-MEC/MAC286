Lecture 1: Introduction to Data Structures

🎯 What Are Data Structures?

📚 Definition

Data Structure: A specialized format for organizing, processing, retrieving, and storing data. It's a way of arranging data in computer memory so that it can be accessed and modified efficiently.

Think of data structures as containers that hold your data in a specific arrangement. Just like you organize your clothes in a closet, books on a shelf, or files in a cabinet, we organize data in computer memory using different structures.

🤔 Why Do We Need Data Structures?

  • Efficiency: Choose the right structure for fast data access
  • Organization: Keep data neat and accessible
  • Memory Management: Use computer memory effectively
  • Problem Solving: Different problems need different approaches

🌟 Real-World Analogy

Imagine you're organizing a library:

  • Dictionary: Books arranged alphabetically (like hash tables)
  • Card Catalog: Index system (like binary search trees)
  • Book Stack: Last book on top (like stacks)
  • Checkout Line: First person served first (like queues)

🔢 Arrays: Our First Data Structure

📚 Array Definition

Array: A collection of elements stored in contiguous memory locations. Each element can be accessed using an index (position number).

Key Characteristics of Arrays:

  • Fixed Size: Size is determined at creation
  • Homogeneous: All elements are of the same data type
  • Random Access: Can access any element directly using its index
  • Zero-Indexed: First element is at position 0

⚙️ Array Operations

1. Traversal (Visiting all elements)

// Print all elements in an array
for(int i = 0; i < numbers.length; i++) {
    System.out.print(numbers[i] + " ");
}
// Output: 10 25 30 45 50

// Enhanced for loop (Java)
for(int num : numbers) {
    System.out.print(num + " ");
}

2. Search (Finding an element)

Linear Search Example

// Search for element 30
public static int linearSearch(int[] arr, int target) {
    for(int i = 0; i < arr.length; i++) {
        if(arr[i] == target) {
            return i; // Return index if found
        }
    }
    return -1; // Element not found
}

// Usage
int position = linearSearch(numbers, 30);
System.out.println("Element found at index: " + position);

3. Insertion (Adding elements)

Insert at specific position

// Insert 77 at position 2
// Original: {10, 25, 30, 45, 50}
public static int[] insertAtPosition(int[] arr, int pos, int value) {
    int[] newArr = new int[arr.length + 1];
    
    // Copy elements before position
    for(int i = 0; i < pos; i++) {
        newArr[i] = arr[i];
    }
    
    // Insert new element
    newArr[pos] = value;
    
    // Copy remaining elements
    for(int i = pos; i < arr.length; i++) {
        newArr[i + 1] = arr[i];
    }
    
    return newArr;
    // Result: {10, 25, 77, 30, 45, 50}
}

4. Deletion (Removing elements)

Delete from specific position

// Delete element at position 1
// Original: {10, 25, 30, 45, 50}
public static int[] deleteAtPosition(int[] arr, int pos) {
    int[] newArr = new int[arr.length - 1];
    
    // Copy elements before position
    for(int i = 0; i < pos; i++) {
        newArr[i] = arr[i];
    }
    
    // Copy elements after position
    for(int i = pos + 1; i < arr.length; i++) {
        newArr[i - 1] = arr[i];
    }
    
    return newArr;
    // Result: {10, 30, 45, 50}
}

⏱️ Time Complexity Analysis

📚 What is Time Complexity?

Time Complexity: A computational complexity that describes the amount of computer time taken by an algorithm to run, as a function of the size of the input. We use Big O notation to express this mathematically.

🔍 Understanding Big O Notation

Big O describes the worst-case scenario - how an algorithm performs when given the most challenging input of size n.

Common Big O Complexities

  • O(1): Constant - Always same time
  • O(log n): Logarithmic - Very efficient
  • O(n): Linear - Grows with input
  • O(n²): Quadratic - Slow for large input

Growth Rate Comparison

For n = 1000:

  • O(1) = 1 operation
  • O(log n) = ~10 operations
  • O(n) = 1,000 operations
  • O(n²) = 1,000,000 operations!

🎯 Array Operations Time Complexity Deep Dive

1. Access by Index - O(1)

// Direct access using calculated memory address
int element = arr[5]; // Always takes same time!

// Formula: address = base_address + (index × element_size)
// Example: address = 1000 + (5 × 4) = 1020

Why O(1)? The computer calculates the exact memory location mathematically, regardless of array size.

2. Linear Search - O(n)

// Worst case: element is at the end or not found
for(int i = 0; i < arr.length; i++) {
    if(arr[i] == target) return i;
}
// Might check all n elements

Why O(n)? In worst case, we must examine every element in the array.

3. Insertion/Deletion in Middle - O(n)

// Insert at position 2 in array of size n
// Must shift (n-2) elements to the right
for(int i = arr.length - 1; i > pos; i--) {
    arr[i] = arr[i - 1]; // Shifting operation
}

Why O(n)? Must shift up to (n-1) elements to make/close space.

Search O(1) O(n) O(n) Element at start vs end/not found
Insert at End O(1) O(1) O(1) No shifting needed
Insert at Start/Middle O(n) O(n) O(n) Must shift existing elements
Delete at End O(1) O(1) O(1) Just reduce size counter

⚖️ Arrays: Pros and Cons

✅ Advantages

  • Fast Access: O(1) random access by index
  • Memory Efficient: No extra memory for pointers
  • Cache Friendly: Elements stored contiguously
  • Simple: Easy to understand and implement
  • Mathematical Operations: Easy to perform bulk operations

❌ Disadvantages

  • Fixed Size: Cannot resize after creation
  • Expensive Insertion/Deletion: O(n) for middle operations
  • Memory Waste: Must allocate for maximum possible size
  • Homogeneous: Can only store one data type
  • No Built-in Bounds Checking: Can access invalid indices

🎯 When to Use Arrays?

Perfect Use Cases

  • Mathematical Computations: Matrix operations, scientific calculations
  • Lookup Tables: Store predefined values for quick access
  • Buffers: Temporary storage for data processing
  • Image Processing: Pixel data representation
  • Sorting Algorithms: When you need to rearrange data

Avoid Arrays When

  • You need frequent insertions/deletions in the middle
  • The data size varies significantly
  • You need to store different data types together
  • Memory usage needs to be minimized
